# bootstrap-confidence-intervals Specification

## Purpose

Quantify uncertainty in hazard concentration and hazard proportion
estimates through parametric or non-parametric bootstrap resampling of the
fitted distributions, with consistent defaults across `ssd_hc()`,
`ssd_hp()` and `predict()`.

## Requirements

### Requirement: Opt-in confidence interval computation
Confidence intervals SHALL be computed only when the caller sets
`ci = TRUE`; the default `ci = FALSE` SHALL produce only point estimates,
with `se`, `lcl` and `ucl` set to `NA` and `nboot` reported as `0` in the
output.

#### Scenario: Default ci is FALSE
- **WHEN** `ssd_hc(fits)` is called with default arguments
- **THEN** the returned tibble has `se`, `lcl` and `ucl` equal to `NA` and
  `nboot` equal to `0`

#### Scenario: Opt-in ci
- **WHEN** `ssd_hc(fits, ci = TRUE)` is called
- **THEN** the returned tibble has numeric `lcl` and `ucl` columns whose
  values are the quantiles of the bootstrap distribution at the stated
  `level`

### Requirement: Confidence level
The `level` argument SHALL control the width of the two-sided confidence
interval and SHALL default to `0.95`, producing the 2.5% and 97.5%
quantiles of the bootstrap distribution in `lcl` and `ucl`.

#### Scenario: Default level
- **WHEN** `ssd_hc(fits, ci = TRUE)` is called with default `level`
- **THEN** `lcl` is the 2.5th percentile and `ucl` is the 97.5th percentile
  of the bootstrap replicates, and the `level` column equals `0.95`

#### Scenario: Custom level
- **WHEN** `ssd_hc(fits, ci = TRUE, level = 0.9)` is called
- **THEN** `lcl` and `ucl` correspond to the 5th and 95th percentiles, and
  the `level` column equals `0.9`

### Requirement: Number of bootstrap iterations
The `nboot` argument SHALL control how many bootstrap replicates are drawn
(default `1000`); `nboot = 0` SHALL skip confidence interval computation
entirely even if `ci = TRUE`.

#### Scenario: Default nboot
- **WHEN** `ssd_hc(fits, ci = TRUE)` is called with default arguments
- **THEN** 1000 bootstrap replicates are drawn and `nboot` in the returned
  tibble reports `1000`

#### Scenario: nboot = 0 short-circuits CI computation
- **WHEN** `ssd_hc(fits, ci = TRUE, nboot = 0)` is called
- **THEN** the function returns without drawing any bootstrap samples and
  the `se`, `lcl` and `ucl` columns are `NA`

### Requirement: Parametric and non-parametric bootstrap
The `parametric` argument SHALL choose between parametric bootstrap
(`parametric = TRUE`, default; replicates are drawn from each fitted
distribution) and non-parametric bootstrap (`parametric = FALSE`;
replicates are drawn by resampling the observed data with replacement).
Censored data SHALL require `parametric = FALSE`.

#### Scenario: Parametric bootstrap
- **WHEN** `ssd_hc(fits, ci = TRUE, parametric = TRUE)` is called on
  uncensored data
- **THEN** each replicate is generated by simulating a new sample of the
  same size from each fitted distribution, refitting, and computing HCx

#### Scenario: Non-parametric bootstrap
- **WHEN** `ssd_hc(fits, ci = TRUE, parametric = FALSE)` is called
- **THEN** each replicate is generated by resampling rows of the original
  data with replacement, refitting, and computing HCx

#### Scenario: Parametric bootstrap on censored data is rejected
- **WHEN** `ssd_hc(fits, ci = TRUE, parametric = TRUE)` is called and the
  input data are censored
- **THEN** the function raises an error directing the caller to use
  `parametric = FALSE`

### Requirement: Bootstrap success rate threshold
The `min_pboot` argument (default `0.8`) SHALL set the minimum proportion
of bootstrap iterations that must succeed (a replicate's distributions all
fit and produce a finite estimate) for a confidence interval to be
returned; when the observed `pboot` is below `min_pboot`, the function
SHALL emit a warning and SHALL return `NA` in `lcl` and `ucl`.

#### Scenario: Successful bootstrap
- **WHEN** at least `min_pboot` of the `nboot` replicates succeed
- **THEN** `lcl` and `ucl` are the requested quantiles of the successful
  replicates and `pboot` reports the success fraction

#### Scenario: Insufficient successful replicates
- **WHEN** fewer than `min_pboot` of the replicates succeed
- **THEN** `lcl` and `ucl` are `NA`, `pboot` reports the actual success
  fraction, and a warning is emitted

### Requirement: Choice of model-averaging confidence-interval method
The `ci_method` argument SHALL select how per-distribution bootstrap
replicates are combined when `average = TRUE`, and SHALL default to
`"weighted_samples"`. Allowed values SHALL include at least
`"weighted_samples"`, `"multi_fixed"`, `"multi_free"`,
`"arithmetic_samples"`, `"geometric_samples"`, `"MACL"` and `"GMACL"`.

#### Scenario: Default ci_method
- **WHEN** `ssd_hc(fits, ci = TRUE, average = TRUE)` is called without a
  `ci_method` argument
- **THEN** the effective `ci_method` is `"weighted_samples"` and the
  `ci_method` column of the returned tibble reports this value

#### Scenario: Invalid ci_method
- **WHEN** `ci_method` is not one of the documented values
- **THEN** the function raises an error

### Requirement: Optional persistence of bootstrap samples
The `save_to` argument SHALL, when non-`NULL`, cause bootstrap replicate
estimates to be written to disk in the supplied directory; when `NULL`
(default) no files SHALL be written. The `samples` argument SHALL, when
`TRUE`, cause the replicate estimates to be attached as a list column on
the returned tibble.

#### Scenario: save_to default
- **WHEN** `ssd_hc(fits, ci = TRUE)` is called with default `save_to`
- **THEN** no files are written to disk

#### Scenario: samples = TRUE
- **WHEN** `ssd_hc(fits, ci = TRUE, samples = TRUE)` is called
- **THEN** the returned tibble gains a list column of per-row bootstrap
  replicate estimates
